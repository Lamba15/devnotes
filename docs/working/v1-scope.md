# V1 Scope

## Position

This file describes the current intended scope of v1.

It is a working scope document, not a permanent product-definition file.

## V1 Focus

The first part of the system to build is:

1. the structure of the system
2. the client domain
3. client project management
4. transactions
5. invoices

## Tracking In V1

Within the client/project area, the known tracking subsystem includes:

- projects
- boards
- issues

## V1 Boundary

V1 does not attempt to define or build the full OS.

V1 is limited to:

- foundational system structure
- client-facing scope
- client project management
- transactions
- invoices

Other future parts of the OS are intentionally outside v1.
